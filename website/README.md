# Project Website

The official web interface for the MedAI Fracture Detection System. Built with **Next.js 16**, React 19, Tailwind CSS, Shadcn UI, and Radix UI.

**Live Demo:** [cvpr-submission-frac-mas.vercel.app](https://cvpr-submission-frac-mas.vercel.app)

## Pages

| Route       | Description                                                                                  |
| ----------- | -------------------------------------------------------------------------------------------- |
| `/`         | Landing page with system overview and features                                               |
| `/diagnose` | Upload X-ray, view diagnosis results, chat with Patient Interface Agent, download PDF report |
| `/rubric`   | Human validation rubric interface                                                            |

## Local Development

1. **Install Dependencies**

   ```bash
   npm install
   ```

2. **Set Environment Variables** — create `.env.local`:

   ```
   BACKEND_URL=https://your-hf-space.hf.space
   ```

3. **Run Development Server**

   ```bash
   npm run dev
   ```

   Open [http://localhost:3000](http://localhost:3000) in your browser.

## Backend Connection

This frontend requires the FastAPI backend in `../backend_hf` to be running or deployed.

1. Deploy `../backend_hf` to a Hugging Face Space (Docker SDK).
2. Point `BACKEND_URL` at the Space URL.

The frontend communicates with three backend endpoints:

| Endpoint         | Used by                                                                                    |
| ---------------- | ------------------------------------------------------------------------------------------ |
| `POST /diagnose` | `/diagnose` page — inference, Grad-CAM, conformal set, Critic verdict, educational summary |
| `POST /chat`     | `/diagnose` page — Patient Interface Agent follow-up chat                                  |
| `POST /report`   | `/diagnose` page — download ReportLab PDF report                                           |

## Backend Feature Notes

- **Conformal prediction:** The backend returns a `conformal_set` (set-valued prediction at a calibrated coverage level). Calibrate the threshold on validation data using `scripts/calibration/prepare_val_and_calibrate.py` and make `conformal_threshold.txt` available to the backend.
- **Stacking ensemble:** A trained stacking pipeline `outputs/stacker.joblib` can be used in place of weighted averaging. Ensure the artifact is present in the backend environment.
- **Per-model Grad-CAM:** The backend generates per-model Grad-CAM overlays when `pytorch-grad-cam` is available. The Explainability panel on `/diagnose` displays these overlays.
- **PDF Reports:** Generated by `report_generator.py` (ReportLab) and streamed as a download from `POST /report`.

Ensure the following artifacts are present on the backend: `models/*.pth`, `models/best.pt`, `outputs/stacker.joblib`, `conformal_threshold.txt`, `chroma_db/`.

## Deployment

This app is optimized for **Vercel**.

1. Push this codebase to GitHub.
2. Import the project into Vercel.
3. Add `BACKEND_URL` as an environment variable in Vercel project settings.
4. Deploy — no additional build configuration required.
